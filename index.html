// Design Principles
What makes this style
🎨
Semantic Color
Color is a language, not decoration. Red = error. Yellow = warning. Green = success. Users learn it once and read page state in seconds without reading text.
𝙼
Monospace as Signal
JetBrains Mono is used only for technical values — numbers, IDs, SQL, timestamps. Its presence signals: "this is data, not prose." Mixed with a sans-serif UI font for clear hierarchy.
▦
Data Density
No decorative white space. Information is dense but not chaotic — semantic groups are separated by 1px dividers, not large gaps. Users see more per viewport.
◧
Surface Hierarchy
Depth is achieved through background color steps: #0F1117 → #161B27 → #1E2535. No box shadows needed — the layers are visible through contrast alone.
✦
Function over Form
Every element justifies its existence with data or an action. Remove it — something is lost. No hero illustrations, no onboarding animations, no decorative gradients.
⚙
Parsed, not Raw
Instead of dumping raw log output, the interface structures information: model name, error message, SQL line. Cognitive load drops significantly.
